Eye Combine Value — มันคำนวณยังไง?

เอกสาร interactive อธิบาย logic การได้มาซึ่งค่า combinedScore (หรือที่เก็บใน Supabase ว่า eye_combined) — ค่าที่ใช้ตัดสินว่าผู้ใช้ ใส่แว่นกันแดด / มีอะไรบังตา หรือไม่ ระหว่างทำ face liveness

ที่มาของ logic: lib/features/face_liveness/application/utils/eye_occlusion_util.dart · port 1:1 จาก production Kotlin (FaceDetectionUtils.isWearingSunglasses)

⚡ TL;DR — สรุปสั้น ๆ

เราไม่ได้ดูว่า "ตาเปิด/ปิด" — เราดูว่า บริเวณดวงตามืด/เรียบ/ซีด เมื่อเทียบกับ แก้ม หรือเปล่า ถ้าใช่ = น่าจะมีแว่นดำบัง

  1. ตัด ROI 4 กรอบจาก face bounding box: ตาซ้าย, ตาขวา, แก้มซ้าย, แก้มขวา (ใช้สัดส่วนล้วน ๆ ไม่พึ่ง landmark)
  2. รวม pixel ของตาซ้าย+ขวาเป็นก้อนเดียว / แก้มซ้าย+ขวาเป็นก้อนเดียว แล้วคำนวณสถิติ 3 ตัว: ความสว่าง (Y), ความแปรปรวน (stdDev), ความอิ่มสี (saturation)
  3. เทียบความสว่างตากับแก้ม → luminanceRatio
  4. เอา 3 สัญญาณเข้าระบบ ให้คะแนนแบบ bucket บวกกันได้ค่า 0.00 – 1.00
  5. ถ้าคะแนนรวม ≥ 0.30 → ตัดสินว่า บังตา (occluded)

1. Pipeline ภาพรวม

1
รับ frame — RGBA8888 ที่หมุนตั้งตรงแล้ว + faceBox (กรอบหน้า)
ถ้า frame เป็น front camera ที่ถูก mirror → flip กรอบกลับก่อน
2
เช็คขนาดหน้า — ถ้าหน้าเล็กกว่า 40×40 px หลัง clip → คืน combinedScore = 0, ไม่บัง
3
ตัด ROI 4 กรอบ จากสัดส่วนของ faceBox (ดูแผนภาพข้อ 2)
4
คำนวณสถิติ pixel — meanY, stdY, meanSat ของก้อนตา (รวม) และก้อนแก้ม (รวม)
5
luminanceRatio = ความสว่างตา ÷ ความสว่างแก้ม
6
bucket scoring — รวม 3 สัญญาณเป็น combinedScore (0–1)
7
ตัดสิน — occluded = combinedScore ≥ 0.30 → ถ้า true ส่ง LivenessFailure.eyeOccluded

2. ROI — ตำแหน่งกรอบบนใบหน้า

ทุกกรอบคำนวณจากสัดส่วนของ faceBox (กว้าง = fw, สูง = fh) — ไม่ใช้ landmark เลย จุดสำคัญคือ แก้มเป็นค่าอ้างอิง (reference) เพราะเป็นผิวหน้าที่ปกติไม่ถูกบัง

Eye ROI — บริเวณที่จะตรวจว่าถูกบัง Cheek ROI — ผิวอ้างอิง (reference)

3. สถิติ pixel 3 ตัว ที่คำนวณต่อ ROI

ไล่ทุก pixel ใน ROI แล้วสะสมค่า:

Y (luminance, ITU-R BT.601) = 0.299·R + 0.587·G + 0.114·B Sat (HSV style) = (max(R,G,B) − min(R,G,B)) / max(R,G,B) × 255 meanY = ค่าเฉลี่ย Y → ตา "มืด" แค่ไหน stdY = ส่วนเบี่ยงเบนมาตรฐาน Y → ตา "เรียบ/แบน" แค่ไหน (เลนส์ทึบ = เรียบ) meanSat= ค่าเฉลี่ย Sat → ตา "ซีด/ไม่มีสี" แค่ไหน

🔅 luminanceRatio

eyeStats.meanY / cheekStats.meanY

ตามืดเทียบกับแก้ม → ค่าต่ำ = น่าสงสัยว่าบัง

📉 stdDev (stdY)

ความแปรปรวนของแสงในกรอบตา

เลนส์ดำทึบ = สีเรียบ = std ต่ำ

🎨 saturation

meanSat ของกรอบตา

เลนส์เทา/ดำ = สีจาง = sat ต่ำ

🛡️ guard

cheekStats.meanY ≤ 1.0

ถ้าแก้มมืดผิดปกติ (หารด้วย ~0) → คืนค่าว่าง ไม่ตัดสิน
